# 如何找到各个功能

## 🎯 添加图表按钮位置

### 方法1：创建新笔记
```
1. 访问 http://localhost:3000/notes
2. 点击右上角"写笔记"按钮
3. 进入编辑页面后，查看顶部操作栏
4. 按钮位置：[编辑] [预览] [📊 添加图表]
```

### 方法2：编辑现有笔记
```
1. 访问 http://localhost:3000/notes  
2. 点击任意笔记卡片
3. 点击右上角"编辑"按钮
4. 在顶部操作栏找到 [📊 编辑图表] 按钮
```

### 按钮外观
- **图标**: 📊 流程图图标 (FaProjectDiagram)
- **文字**: 
  - 未添加图表时："添加图表"
  - 已有图表时："编辑图表"
- **颜色**:
  - 未添加：灰色背景
  - 已添加：紫色背景（突出显示）
- **位置**: 紧跟在"预览"按钮之后

## 📍 字体大小调整位置

### 查看笔记页面
```
/notes/view/:id
→ 顶部操作栏
→ [返回笔记列表] ... [- 📏默认 +] [编辑] [删除]
```

### 编辑器预览模式
```
/notes/editor
→ 点击"预览"
→ 顶部出现 [- 📏默认 +]
```

### 按钮外观
- **图标**: 
  - 左：FaSearchMinus (-)
  - 中：FaTextHeight (📏)
  - 右：FaSearchPlus (+)
- **文字**: 小/默认/中/大/特大
- **背景**: 灰色圆角容器
- **位置**: 编辑和删除按钮之前

## 🔍 如果找不到按钮

### 检查清单

#### 1. 确认页面正确
```bash
✅ 是在 /notes/editor 页面吗？
✅ 不是在 /notes 首页吗？
✅ 开发服务器正在运行吗？
```

#### 2. 检查浏览器窗口大小
```
如果窗口太小，按钮可能折行
→ 尝试最大化窗口
→ 或横向滚动查看
```

#### 3. 刷新页面
```bash
# 硬刷新
Ctrl + F5 (Windows)
Cmd + Shift + R (Mac)
```

#### 4. 查看控制台错误
```
按 F12 打开开发者工具
→ 查看 Console 标签
→ 看是否有错误信息
```

## 📱 不同屏幕大小的布局

### 大屏幕 (>1024px)
```
[编辑] [预览] [添加图表] [- 📏默认 +] ... [取消] [保存]
所有按钮在一行
```

### 中等屏幕 (768-1024px)
```
[编辑] [预览] [添加图表]
[- 📏默认 +] ... [取消] [保存]
可能折成两行
```

### 小屏幕 (<768px)
```
[编辑] [预览]
[添加图表]
[- 📏 +]
[取消] [保存]
多行排列
```

## 💡 使用提示

### 第一次使用
```
1. 先写一些笔记内容
2. 然后点击"添加图表"
3. 在流程图编辑器中设计图表
4. 点击"保存"
5. 图表嵌入到笔记中
6. 点击"保存笔记"完成
```

### 按钮颜色含义
- **灰色** - 尚未添加图表
- **紫色** - 已有图表（点击可编辑）
- **蓝色** - 当前选中的标签页

### 功能优先级
```
高优先级：
- 编辑/预览切换 ← 基础功能
- 保存笔记 ← 必须操作

中优先级：
- 添加图表 ← 增强功能
- 字体调整 ← 阅读优化

低优先级：
- 算法特定字段 ← 特定分类
```

## 🎯 故障排除

### 问题：看不到"添加图表"按钮

**可能原因1：不在编辑页面**
```
解决：确认URL是 /notes/editor
```

**可能原因2：代码未更新**
```bash
# 停止服务器 (Ctrl+C)
# 重新启动
npm start
```

**可能原因3：浏览器缓存**
```
硬刷新：Ctrl + F5
或清除缓存后刷新
```

**可能原因4：屏幕太小**
```
缩放浏览器：Ctrl + "-" 缩小显示
或最大化窗口
```

### 问题：按钮是灰色的

**这是正常的！**
```
灰色 = 未添加图表
点击后会打开流程图编辑器
添加图表后按钮变紫色
```
