## 📐 比例分配

### 桌面端
- **天气系统**: 2/3 宽度（约66.7%）
- **日历系统**: 1/3 宽度（约33.3%）
- **展示窗**: 全宽

### 设计理由
1. 天气信息更丰富，需要更多空间
2. 日历使用紧凑设计，节省空间
3. 充分利用屏幕空间
4. 平衡美观和实用性

## 🎯 组件对比

### 天气系统（大版）
- 位置：左侧 2/3 宽度
- 特点：
  - ✅ 完整的天气信息
  - ✅ 24小时预报
  - ✅ 5天预报
  - ✅ 生活指数
  - ✅ 可展开查看详情

### 日历系统（紧凑版）
- 位置：右侧 1/3 宽度
- 特点：
  - ✅ 月历网格显示
  - ✅ 今日事件列表
  - ✅ 快速添加事件
  - ✅ 事件标记点
  - 🔄 简化了详细视图

### 展示窗（全宽）
- 位置：下方全宽
- 特点：
  - ✅ 轮播展示
  - ✅ 快速访问
  - ✅ 内容推荐

## 🔄 与旧布局对比

### 旧布局
```
天气：无
日历：2/3 宽度（很大）
音乐：右侧说明框 1/3 宽度
```

### 新布局
```
天气：2/3 宽度（主要）
日历：1/3 宽度（紧凑）
音乐：固定侧边栏（不占主体空间）
```

## ✨ 优势

1. **信息更丰富**: 
   - 新增天气系统
   - 保留日历功能
   - 音乐播放器更方便

2. **空间利用**: 
   - 天气占大头，显示更多信息
   - 日历紧凑但功能完整
   - 固定播放器不占主体空间

3. **用户体验**:
   - 一屏看到天气、日程、音乐
   - 信息密度适中
   - 操作便捷

4. **响应式**:
   - 桌面端并排显示
   - 移动端垂直堆叠
   - 自动适配屏幕

## 📱 响应式断点

```css
/* 移动端 */
grid-cols-1 (< 1024px)
  └─ 天气、日历垂直堆叠

/* 桌面端 */
lg:grid-cols-3 (≥ 1024px)
  ├─ 天气: lg:col-span-2 (2/3)
  └─ 日历: lg:col-span-1 (1/3)
```

## 🔧 自定义比例

### 调整天气/日历比例

#### 方式1: 均分（各占1/2）
```javascript
<div className="lg:col-span-1">  // 天气
<div className="lg:col-span-1">  // 日历
```

#### 方式2: 当前设计（2/3 + 1/3）
```javascript
<div className="lg:col-span-2">  // 天气
<div className="lg:col-span-1">  // 日历
```

#### 方式3: 天气占主导（3/4 + 1/4）
```javascript
<div className="lg:col-span-3">  // 天气
<div className="lg:col-span-1">  // 日历
```
然后将grid改为 `grid-cols-4`
